Key Principles

  • Visibility
  • Consistancy
  • Feedback
  • Constraints
  • Affordance
  • Accessability

Complexity Consideration

  • Website should be simple to Navigate not too simple
  • Easy to uunderstand features quickly
  • Be able to remeber the features and use them effectivly
  • Use Icons
  • The user should learn by doing
  • Should be as easy as Email
  • Know functionality for application
  • Guide people to functionality

Emotional Principles

  • Use specific colors to make the site fell more welcoming
  • Avoid really bright colors that might hurt the eyes
  • Use professonal fonts for websites, do not use fonts that might hurt the eyes

Cultrual Consideration

  • Watch Language and symbols used
  • Make sure the website is orgnaized
  • Do not assume two place close have same culture
  • Culture focuses on long term or short term

Conceptional Consideration

  • Design our model and metapor not from technology
  • Don't use the wrong model
  • Make sure mental & conceptual match

Accessability Principles

  • The use of multiple visual ques
  • Have clear instances for input
  • Have option to read text outloud
  • Have alternete text on graphics when needed

Interface Principles

  • Dont make a user think
  • Have user significance
  • Have user constraints
  • Have organized areas
  • Take advantage of convention
  • Avoid distractions
  • Have actating logo
  • Make it highly astetic and highly usuable
  • Have consistancy
  • Keep text visiable
  • Buttons in comfotable placese

  • Mobile Devices

  • Make sure everything is visible
  • Nice and easy navigation
  • Allow for easy scanning for users
  • check for Accessability for both laptop and mobile
  • Allow for zooming
  • Have a cursor
  • Have a strong affordance for the smaller display
